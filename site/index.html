Why shielded payments
Agents leak payment data

When AI agents pay on public chains, every transaction reveals who called what API, how often, and how much they spent. That's a competitive intelligence goldmine. Zcash fixes this.

Usage fingerprinting

Public payments let competitors see exactly which APIs your agent uses, how frequently, and spending patterns.

Price discrimination

When services can see your total spend across APIs, they can adjust pricing based on your usage profile.

Encrypted verification

Servers verify payments using an Orchard incoming viewing key. Only the recipient can decrypt and confirm — no one else.

Accept private payments in 5 minutes

Drop zimppy into your existing API server. TypeScript or Rust — same HTTP 402 flow, same MPP spec, fully shielded underneath.

TypeScript Server

mppx + zimppy-ts 
import { Mppx } from 'mppx/server'
import { zcash } from 'zimppy-ts'

const payment = Mppx.create({
  methods: [zcash({
    orchardIvk: config.orchardIvk,
    rpcEndpoint: 'https://zebrad.example.com',
  })],
  secretKey: 'your-hmac-secret',
})

// In your handler:
const result = await payment.charge({
  amount: '10000', currency: 'zec',
  recipient: config.address,
  methodDetails: { network: config.network },
})(request)

if (result.status === 402) return result.challenge
return result.withReceipt(Response.json({ data }))

use zimppy_rs::ZcashChargeMethod;

let charge = ZcashChargeMethod::new(

// In your axum handler:
match charge.verify_payment(

        // outcome.observed_amount_zat

        (StatusCode::OK, Json(data)).into_response()
    }
    _ => issue_challenge(state).await,
}
How it works
Standard HTTP 402,
shielded underneath

Same MPP flow every agent already speaks. The server returns a 402 challenge, the agent pays with shielded ZEC, retries with proof, and gets the response. One HTTP round-trip, fully private.

CLI
npx zimppy

Wallet creation, payments, sessions — one command. No config files, no env vars. Works with any MPP-enabled API.

Create a wallet

Fresh Zcash keys, ready in seconds. Named wallets for multi-identity.

$ npx zimppy wallet create

Pay for an API call

Auto-discovers pricing, sends shielded payment, retries with credential.

$ npx zimppy request https://api.example.com/fortune

Sessions

Deposit once, instant requests. Close to get unused balance refunded.

$ npx zimppy session close

Multiple wallets

Separate identities for different agents or environments.

$ npx zimppy wallet use work
