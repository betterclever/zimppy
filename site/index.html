Why shielded payments
Agents leak payment data

When AI agents pay on public chains, every transaction reveals who called what API, how often, and how much they spent. That's a competitive intelligence goldmine. Zcash fixes this.

Usage fingerprinting

Public payments let competitors see exactly which APIs your agent uses, how frequently, and spending patterns.

Price discrimination

When services can see your total spend across APIs, they can adjust pricing based on your usage profile.

Encrypted verification

Servers verify payments using an Orchard incoming viewing key. Only the recipient can decrypt and confirm — no one else.

Server

import { Mppx } from 'mppx/server'
import { zcash } from 'zimppy-ts/server'

const mppx = Mppx.create({
  methods: [await zcash({ wallet: 'server' })],
})

const result = await mppx.charge({
  amount: '42000',
  currency: 'zec',
})(request)

if (result.status === 402) return result.challenge
return result.withReceipt(Response.json({ data }))
use mpp::server::axum::*;
use mpp::server::Mpp;
use zimppy_rs::ZcashChargeMethod;

struct FortunePrice;
impl ChargeConfig for FortunePrice {
  fn amount() -> &'static str { "42000" }
}

let mpp = Mpp::create(ZcashChargeMethod::new(
  &rpc, &address, &orchard_ivk,
))?;
let state: Arc<dyn ChargeChallenger> =
  Arc::new(mpp);

let app = Router::new()
  .route("/api/fortune", get(fortune))
  .with_state(state);

async fn fortune(
  charge: MppCharge<FortunePrice>,
) -> WithReceipt<Json<Value>> {
  WithReceipt {
    receipt: charge.receipt,
    body: Json(json!({ "fortune": pick() })),
  }
}

Client

import { Mppx } from 'mppx/client'
import { zcash } from 'zimppy-ts/client'

const mppx = Mppx.create({
  methods: [zcash({ wallet: 'default' })],
})

// 402 → pay → retry handled automatically
const res = await mppx.fetch('https://api.example.com/fortune')
console.log(await res.json())
use mpp::client::Fetch;
use zimppy_rs::ZcashPaymentProvider;

let provider = ZcashPaymentProvider::new(
  wallet_config, &rpc_endpoint,
);

// 402 → pay → retry handled automatically
let resp = client
  .get("https://api.example.com/fortune")
  .send_with_payment(&provider)
  .await?;

let fortune = resp.json::<Fortune>().await?;

CLI
npx zimppy

Agent-native wallet. Full MPP spec support. One command to pay any 402 endpoint.

Wallet identity

Shielded address, balance, network status. Ready for agents.

$ npx zimppy wallet whoami

MPP charge + session

Auto-discovers 402 pricing, handles charge or session flow, retries with credential.

$ npx zimppy request https://fortune.api.dev/fortune

Shielded transfers

Send ZEC with full shielded privacy. Sender, receiver, amount all encrypted.

$ npx zimppy wallet send <addr> 42000

Multi-agent wallets

Isolated identities per agent, environment, or task. Switch in one command.

$ npx zimppy wallet use work
